# 预测数据集

工业大模型有一个根据当前数据预测未来走势的功能，大概的逻辑是，上传尽可能多的数据（时间间隔5秒， 总共10000点），然后可以预测未来10分钟的关键数据的走势；大模型根据历史数据中的一些横向（位号与位号之间）和纵向（时间序列上）关联，找到数据之间的联系，并按照最新的数据，推演出未来的数据。


既然是根据历史经验的数据预测，那么目前可以考虑到的底层逻辑大概有以下几层

1. 某些数据会随着时间，进行有规则地变化（如光照强度，会跟随这每天的日出日落变化）；
2. 某些数据会跟随着其他参数，形成一些滞后的线性跟随（如热水器的水温，会随着电功率呈现一个滞后的上涨）；
3. 某些数据会由其他多个参数，有一些多项式关系（如 a = kx + ly + mz + n）
4. 以上基本逻辑的无限组合

我们构建数据时，会从简单到复杂，从单一到组合，逐步对产品进行验证；
由于我们构建的是10000点，那么本质上，应该是构建10000点+10分钟(10 * 60 / 5 = 120) = 10120点，并且希望最后这120点的数据是存在特征变化的
